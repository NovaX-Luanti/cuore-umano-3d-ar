Fase 1 – Pianificazione del Progetto

Obiettivo: realizzare un’esperienza WebAR che mostrasse un cuore umano realistico, accessibile a studenti e visitatori tramite un semplice QR. Ruoli divisi in tre micro-team (Contenuti, Dev, Comunicazione) e timeline di 7 giorni tracciata su GitHub Projects.

Fase 2 – Preparazione dei Materiali

Il modello “Realistic Human Heart” (licenza CC-BY) è stato scaricato da Sketchfab in formato .glb già ottimizzato (< 90 k triangoli, texture 4 K). Nessuna modifica con Blender: l’asset originale garantiva la massima qualità su mobile senza ulteriori operazioni.

Fase 3 – Realizzazione WebAR & Codice

Embed del modello

Tutto è stato sviluppato con puro HTML + CSS + JavaScript vanilla. Il cuore viene visualizzato tramite l’iframe di Sketchfab, che include già il pulsante AR. 

<iframe id="appFrame"
  allow="autoplay; fullscreen; xr-spatial-tracking"
  src="https://sketchfab.com/models/3f8072336ce94d18b3d0d055a1ece089/embed">
</iframe>
  • allow="xr-spatial-tracking" abilita WebXR su Android.
  • Sketchfab genera automaticamente il file .usdz per AR Quick Look su iOS.

Loader animato

Prima che l’iframe sia pronto, mostriamo un cuore <i class="fa-solid fa-heart"> che pulsa con l’animation @beat e frasi dinamiche (“Ossigenazione in corso…”). 

const phrases = [
  'Caricamento dei battiti…',

  'Avvio della circolazione…'
];
let i = 0;
setInterval(() => {
  phraseEl.textContent = phrases[i = (i+1) % phrases.length];
}, 3000);

Controllo offline

Se la connessione cade, sostituiamo loader/iframe con un’icona Wi-Fi barrata grazie agli eventi online/offline: 

window.addEventListener('offline', () => transition(loader, offlineNotice));
window.addEventListener('online',  () => transition(offlineNotice, content));

Navigazione a schede

La funzione showSection() gestisce il passaggio fra le sezioni della pagina con un effetto blur-in/out da 14 px: 

function showSection(id, link){
  const current = document.querySelector('section.active');
  const next    = document.getElementById(id);
  if(!next || current === next) return;

  document.querySelectorAll('nav a').forEach(a => a.classList.remove('active'));
  link.classList.add('active');

  current.classList.add('blur-out');
  setTimeout(() => {
    current.classList.remove('active','blur-out');
    next.classList.add('active','blur-in');
    setTimeout(() => next.classList.remove('blur-in'), 600);
  }, 600);
}

Nessun framework, nessuna pipeline Unity: solo tre file (index.html, style interno e uno <script>) commitati su GitHub Pages per un deploy immediato.

Fase 4 – Generazione del QR Code

Il QR è creato con qrgen-icon (node qrgen-icon.js --url ... --logo ...) e include il logo NovaX al centro. Output in PNG retina e correzione H per una scansione stabile fino a 5 m.

Fase 5 – Contenuti Didattici

Schede PDF, quiz Google Forms, infografiche SVG animate: l’esperienza unisce visualizzazione 3D, spiegazioni testuali e verifica immediata delle conoscenze.

Fase 6 – Test e Presentazione

Test riusciti su iPhone 11, Redmi Note 13 Pro+, Chromebook e laptop Windows: tempo medio di avvio < 5 s in 4 G. Allo stand: poster A1 con QR, tablet di prova, monitor che riproduce il modello.
